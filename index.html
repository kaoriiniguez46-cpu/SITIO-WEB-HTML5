¿Qué es HTML5?

HTML5 es la versión actual para crear sitios web, con nuevas etiquetas, soporte multimedia y mejoras en accesibilidad, sirve para crear sitios web y aplicaciones móviles,y mejorar la accesibilidad,es importante porque es el estándar para crear sitios web accesibles, compatibles y modernos.

Estructura básica de HTML5

<!DOCTYPE html> <html> <head> <title>Mi página</title> </head> <body> </body> </html>

doctype: Es una declaración que indica al navegador la versión de HTML que se está utilizando (en este caso, HTML5). Su función es asegurar que el navegador interprete correctamente el contenido del documento, aplicando las reglas y estándares correspondientes en lugar de un modo de compatibilidad obsoleto.

html: Es el elemento raíz que engloba todo el contenido de la página web. Actúa como contenedor principal y establece el idioma del documento a través del atributo lang, facilitando la accesibilidad y el posicionamiento en buscadores.

head: Contiene información no visible para el usuario en la página, pero es esencial para el funcionamiento y la presentación del contenido. Incluye elementos como title (título de la pestaña del navegador),meta (datos sobre el documento, como codificación o descripción), enlaces a hojas de estilo CSS, scripts JavaScript y recursos externos.

body: Contiene todo el contenido visible de la página web, como texto, imágenes, enlaces, tablas, formularios y otros elementos interactivos. Es la parte que los usuarios ven y con la que interactúan, y su estructura define la organización visual del sitio.

Etiquetas principales

h1 - h6

Encabezados de nivel 1 (más relevante) a 6 (menos relevante) para estructurar texto.

p

Define un párrafo de texto

img

Muestra imágenes; requiere src (ruta) y alt (texto alternativo).

a

Crea enlaces a otras páginas, archivos o secciones (usa el atributo href).

button

Se utiliza para crear elementos interactivos de botón que permiten a los usuarios realizar acciones en una página web.

ul / ol / li

ul - Lista desordenada. Define una lista de elementos sin un orden específico, que por defecto se muestra con viñetas (puntos, círculos o cuadrados). Se usa para contenido donde el orden no importa (ej: lista de compras, ítems de un menú).

ol - Lista ordenada. Crea una lista de elementos con un orden secuencial, que se muestra por defecto con números arabigos, aunque se puede personalizar con letras o números romanos. Se usa cuando el orden es relevante (ej: pasos de un tutorial, ranking).

li- Elemento de lista. Es el elemento que contiene cada ítem de una lista, y debe estar siempre dentro de ul o ol (nunca solo). Puede incluir texto, imágenes, enlaces e incluso otras listas.

HTML5 semántico

header

Es un elemento semántico que define la sección de encabezado de una página o de un componente dentro de ella. Su objetivo es agrupar contenido introductorio o de navegación, mejorando la estructura del código, la accesibilidad y el posicionamiento en buscadores.

nav

Es un elemento semántico que define una sección de navegación, agrupando enlaces principales para moverse por el sitio web o dentro de una página. Su propósito es indicar claramente a navegadores y tecnologías de asistencia qué parte del contenido corresponde a la navegación.

main

Es un elemento semántico que define el contenido principal y único de la página web, es decir, el material directamente relacionado con el tema central o la funcionalidad principal del sitio.

section

Es un elemento semántico que define una sección temática dentro de un documento o aplicación. Su propósito es agrupar contenido relacionado que se centra en un mismo tema o propósito, ayudando a organizar la estructura del sitio de manera clara para navegadores, motores de búsqueda y tecnologías de asistencia.

article

Es un elemento semántico que define un contenido independiente y autocontenido, es decir, que podría ser distribuido o reutilizado por sí solo sin perder su significado (por ejemplo, en un feed de noticias o un listado de artículos).

footer

Es un elemento semántico que define la sección de pie de página de una página o de un componente dentro de ella. Su función es agrupar contenido de cierre o informativo, mejorando la estructura del código y la accesibilidad.

Formularios en HTML5

Los formularios son componentes web que recopilan datos de usuarios para enviarlos a un servidor. Los elementos principales son:

form: Contenedor principal; define la URL de procesamiento (action) y método de envío (GET/POST).

label: Etiqueta descriptiva asociada a campos (via for = id del elemento), mejora accesibilidad.

input: Elemento versátil con tipos como text, email, password, checkbox o radio; usa name, id y required entre otros atributos.

textarea: Campo de texto multilínea; se configura con rows y cols.

select: Menú desplegable; incluye option para cada elección y optgroup para agruparlas.

button: Botón para enviar (type="submit"), restablecer (type="reset") o ejecutar acciones (type="button").

Multimedia en HTML5

HTML5 permite integrar imágenes, audio y video de forma nativa sin plugins externos, con herramientas para compatibilidad y control.

Imágenes

Se usan con img, con atributos como src (ruta), alt (accesibilidad), loading="lazy" (rendimiento) y srcset/sizes (imágenes adaptativas).

Audio

Etiqueta `` con controles nativos (atributo controls). Se usan `` para ofrecer formatos como MP3, WAV o OGG, asegurando compatibilidad. Incluye opciones de autoplay, loop y muted.

Video

Etiqueta video con características similares al audio, además de atributos como width/height (dimensiones) y poster (imagen de portada). Formatos comunes: MP4, WebM, OGG.

Accesibilidad

Permite a lectores de pantalla describir imágenes a usuarios con discapacidad visual; se usa alt="" para imágenes decorativas.

Conclusión

HTML5 representa un hito fundamental en la evolución del desarrollo web, transformando la manera en que se crean, estructuran y consumen sitios y aplicaciones digitales. Al reemplazar tecnologías dependientes de plugins externos y consolidar estándares universales, ha democratizado el acceso a contenido multimedia, permitiendo integrar imágenes, audio y video de forma nativa con mayor eficiencia. Su conjunto de etiquetas semánticas —como header, nav, main, section, article y footer — ha mejorado la estructura del código, facilitando tanto el posicionamiento en buscadores como la accesibilidad para personas con discapacidades, asegurando que la web sea un espacio más inclusivo. Además, la incorporación de formularios robustos y herramientas para el manejo de datos ha simplificado la interacción entre usuarios y plataformas. En definitiva, HTML5 ha sentado las bases para un desarrollo web más moderno, flexible, compatible y centrado en la experiencia del usuario, consolidándose como el lenguaje fundamental que sigue impulsando la innovación en el entorno digital actual y futuro.
